Похоже, таблицы `news` и `news_images` не были созданы в базе данных, поэтому вы получали ошибки `relation "news_images" does not exist` и новости не отображаются.

Давайте проверим список таблиц и, если их нет, создадим их.

Выполните на сервере:

```bash
# 1. Проверяем наличие таблиц
docker exec -i fb-net-db psql -U supabase_admin -d postgres -c "\dt public.*"
```

Если в списке нет `news` и `news_images`, выполните следующую команду для их создания:

```bash
# 2. Создаем таблицы для новостей
docker exec -i fb-net-db psql -U supabase_admin -d postgres < scripts/create-news-tables.sql
```

После этого снова запустите миграцию изображений (так как пути в БД не обновлялись из-за отсутствия таблиц):

```bash
# 3. Повторяем миграцию (чтобы прописать пути к картинкам в БД)
./migrate-images-docker.sh
```
